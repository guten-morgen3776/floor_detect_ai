# ２段目のrefine用モデル

１段目のモデルのエッジ部分だけを切り出して、精緻化モデルに入れる

### 具体的な設計案

具体的なエッジパッチの切り出しと、モデル学習は論文と同じでいい

[arxiv.org](https://arxiv.org/pdf/2104.05239)

ワークフロー

mask2formerで推論→境界を切り出す→軽量のCNNモデルで境界検出→その境界を新たなマスクとして融合

```bash
CubiCasa5k 訓練画像
    │
    ├─ [画像] ──────────────────────────→ image_patch
    │
    ├─ Stage1 (Mask2Former) 推論
    │       ↓
    │   予測マスク群（ふにゃふにゃ含む）
    │       ↓
    │   GTマスクとマッチング (IoU > 0.5)
    │       ↓
    │   予測マスクの境界に沿ってパッチ抽出
    │       ↓
    └─ [予測マスク patch] ──────────────→ mask_patch  (モデル入力)
            ↓
        同じbboxでGTを切り出す
            ↓
        [GTマスク patch] ───────────────→ label
```

学習方法

mask2formerの予測のエッジパッチとGTマスクの両方を入力→その関係性を学習させる
GTマスクはsegformerで使った白黒のマスクでOK

```bash
学習時の1サンプル:

  元画像パッチ          予測マスクパッチ(入力)    GTマスクパッチ(教師)
  ┌──────────┐          ┌──────────┐             ┌──────────┐
  │  建築図面  │    +     │ ふにゃふにゃ│    →学習→   │ ビシッと  │
  │          │          │ な境界    │             │ した境界  │
  └──────────┘          └──────────┘             └──────────┘
      RGB 3ch               mask 1ch                 label
```

モデルは高解像度のままで扱える軽量モデルがいい　
多分１枚あたり100枚程度のパッチを切り出すので、バッチ処理をするなら軽量がいい　
小型のUnetとかがベースラインとしては良さそう　MobileNetV3-Small ベースのU-Net 

## やるべきこと

- 学習データに対して、１段目モデルで推論して予測マスクを作る（4k枚）
- ２段目モデルを作成
- パッチ切り出しの処理を詰める
- 正解マスク（segformer用の白黒マスク画像）と予測マスクを使って２段目モデルを学習
