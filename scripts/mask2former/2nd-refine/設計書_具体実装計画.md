## 0. プロジェクト概要

Mask2Former（exp04-GC）の予測マスクを、BPR（Boundary Patch Refinement）手法で後処理する２段目モデルを構築する。
境界付近のパッチのみを軽量 MobileNetV3-Small U-Net で精緻化することで、大幅な計算コスト増なしに境界品質を改善する。

## 6. dataset.py の設計

### 方針: オンデマンド生成（パッチをディスク保存しない）

400K パッチ × 128×128 × 4ch = 約 25 GB（float32）のディスクを節約するため、
エポックごとに動的にパッチを生成する。その代わり CPU のボトルネックになる可能性があるため
`num_workers >= 4` で DataLoader を回す。

```python
class BoundaryPatchDataset(Dataset):
    """
    1サンプル = 1境界パッチ（image_patch 3ch + pred_mask_patch 1ch, gt_label）

    __init__ では「どの画像のどの予測マスクを使うか」のインデックスリストだけを構築し、
    __getitem__ でパッチ切り出しを実施する。

    インデックス構築:
        for stem in stems:
            load pred_masks, gt_masks
            match pred_mask to gt_mask by IoU > IOT_FILTER_THR
            for each matched (pred_mask, gt_mask) pair:
                boundary = extract_boundary(pred_mask)
                patches_boxes = nms(generate_candidates(boundary))
                for box in patches_boxes:
                    index_list.append((stem, instance_idx, box))

    このリスト構築は __init__ で一度実行（初回 ~数分）→ worker ごとにキャッシュ。
    """

    def __init__(self, img_dir, pred_dir, gt_dir, transform=None, is_train=True):
        self.img_dir   = img_dir
        self.pred_dir  = pred_dir
        self.gt_dir    = gt_dir
        self.transform = transform
        self.is_train  = is_train
        self.samples   = self._build_index()   # list of (stem, inst_idx, box, gt_mask_idx)

    def _build_index(self) -> list:
        ...  # 上記の通り

    def __len__(self):
        return len(self.samples)

    def __getitem__(self, idx):
        stem, inst_idx, box, gt_idx = self.samples[idx]
        x1, y1, x2, y2 = box

        # RGB 読み込み
        image = load_rgb(self.img_dir / f"{stem}.png")

        # 予測マスク・GTマスク 読み込み
        pred_data = np.load(self.pred_dir / f"{stem}_pred.npz")
        pred_mask = pred_data["masks"][inst_idx]   # (H, W) bool
        gt_data   = np.load(self.gt_dir  / f"{stem}.npz")
        gt_mask   = gt_data["masks"][gt_idx]       # (H, W) bool

        # パッチ切り出し & リサイズ
        img_patch  = cv2.resize(image[y1:y2, x1:x2],   (INPUT_SIZE, INPUT_SIZE), interpolation=cv2.INTER_LINEAR)
        mask_patch = cv2.resize(pred_mask[y1:y2, x1:x2].astype(np.float32),
                                (INPUT_SIZE, INPUT_SIZE), interpolation=cv2.INTER_NEAREST)
        gt_patch   = cv2.resize(gt_mask[y1:y2, x1:x2].astype(np.int64),
                                (INPUT_SIZE, INPUT_SIZE), interpolation=cv2.INTER_NEAREST)

        # 正規化
        if self.transform:
            augmented  = self.transform(image=img_patch, mask=mask_patch, gt=gt_patch)
            img_patch  = augmented["image"]   # float32 (3, H, W) after normalize
            mask_patch = augmented["mask"]
            gt_patch   = augmented["gt"]

        # mask 正規化: [0,1] → [-1,1]
        mask_norm = (mask_patch - 0.5) / 0.5

        # 4ch 入力
        # img_patch は albumentation の Normalize で (H,W,3) float → transpose → (3,H,W)
        input_4ch = np.concatenate([
            img_patch.transpose(2, 0, 1),         # (3, H, W)
            mask_norm[np.newaxis, :]               # (1, H, W)
        ], axis=0)                                 # (4, H, W) float32

        return (
            torch.from_numpy(input_4ch),           # (4, 128, 128)
            torch.from_numpy(gt_patch).long()      # (128, 128)  0 or 1
        )
```

### Skip connection チャネルサイズ確認

MobileNetV3-Small の実際の中間出力チャネルは torch 実装により若干異なる場合があるため、
`model.py` 起動時に `_check_encoder_channels()` を呼んで動的にチャネル数を確認する。

```python
def _check_encoder_channels(model_id="mobilenet_v3_small"):
    """
    MobileNetV3-Small の各ステージ出力チャネル数を確認するユーティリティ。
    実際のチャネル数に合わせて dec* の in_ch を調整する。
    """
    import torchvision.models as M
    backbone = M.mobilenet_v3_small(weights=M.MobileNet_V3_Small_Weights.DEFAULT)
    x = torch.zeros(1, 3, 128, 128)
    s0 = backbone.features[0](x)
    s1 = backbone.features[1:3](s0)
    s2 = backbone.features[3:6](s1)
    s3 = backbone.features[6:10](s2)
    bt = backbone.features[10:](s3)
    print(f"s0={s0.shape}, s1={s1.shape}, s2={s2.shape}, s3={s3.shape}, bt={bt.shape}")
```

## 8. step2_train.py の設計

### 学習ループ

```python
# イテレーション基準で管理（epoch 基準でなく）
# 理由: パッチ数が多く1 epoch が非常に長くなるため

model    = LightweightRefineNet(in_channels=4, num_classes=2).to(DEVICE)
optimizer = torch.optim.SGD(model.parameters(),
                            lr=LR, momentum=MOMENTUM, weight_decay=WEIGHT_DECAY)
scheduler = torch.optim.lr_scheduler.LambdaLR(
    optimizer,
    lr_lambda=lambda step: (1 - step / TOTAL_ITERS) ** POLY_POWER
)
criterion = nn.CrossEntropyLoss()

# DataLoader (IterableDataset 的に無限ループさせる)
train_loader = DataLoader(train_dataset, batch_size=BATCH_SIZE,
                          shuffle=True, num_workers=4, pin_memory=True,
                          drop_last=True)
loader_iter = iter(train_loader)

best_val_iou = 0.0

for step in range(TOTAL_ITERS):
    try:
        inputs, targets = next(loader_iter)
    except StopIteration:
        loader_iter = iter(train_loader)
        inputs, targets = next(loader_iter)

    inputs, targets = inputs.to(DEVICE), targets.to(DEVICE)
    logits = model(inputs)         # (B, 2, 128, 128)
    loss   = criterion(logits, targets)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    scheduler.step()

    if (step + 1) % VAL_EVERY_ITERS == 0:
        val_iou = validate(model, val_loader)
        if val_iou > best_val_iou:
            best_val_iou = val_iou
            torch.save(model.state_dict(), BEST_MODEL)
        torch.save(model.state_dict(), LAST_MODEL)
        print(f"Step {step+1}/{TOTAL_ITERS}  val_iou={val_iou:.4f}  best={best_val_iou:.4f}")
```

### Validation 指標

- **パッチ単位 IoU**: 各パッチの foreground IoU を平均
- 完全な mask-level IoU は step3 の end-to-end 評価で測定

## 9. step3_inference.py の設計（End-to-End パイプライン）

```
入力画像 (RGB)
    ↓
Mask2Former 推論（predict04.py と同じ）
    ↓
予測マスク群 (N, H, W) bool
    ↓
for each pred_mask:
    境界抽出 → NMS（閾値=NMS_IOU_INFER）
    ↓
    各パッチを 4ch 入力に変換 → BPR モデル → refined logits
    ↓
    reassemble_patches() で元マスクに統合
    ↓
精緻化済みマスク群 (N, H, W) bool
    ↓
オーバーラップ解消・面積フィルタ（predict04.py と同じ）
    ↓
最終出力保存（PNG / NPZ）
```

## 11. ハイパーパラメータ感度と調整指針

| パラメータ | 推奨初期値 | 調整の方向性 |
|-----------|----------|------------|
| `PATCH_SIZE` | 64 | 固定（論文最適値） |
| `INPUT_SIZE` | 128 | 256 にすると精度 +0.5% 程度、速度 4倍遅い |
| `NMS_IOU_TRAIN` | 0.25 | 固定 |
| `NMS_IOU_INFER` | 0.35 | 0.55 まで上げると精度改善（速度低下） |
| `IOT_FILTER_THR` | 0.5 | 下げると学習が不安定、上げるとデータが減る |
| `BATCH_SIZE` | 64 | A100 なら 128 も試す |
| `TOTAL_ITERS` | 100K | 80K で収束すれば早期停止で OK |
| `LR` | 0.01 | 発散する場合 0.005 に下げる |

## 12. 建築図面特有の考慮点（BPR 設計書より）

1. **壁の線が細い**: `dilation_radius=1` の境界抽出で十分な境界ピクセルが取れるか確認。
   取れない場合は `dilation_radius=2` に変更する。

2. **低テクスチャ**: RGB チャンネルより mask チャンネルへの依存が高くなる。
   mask の正規化（mean=0.5, std=0.5）は必ず適用する。

3. **直線境界**: 論文（自然画像）より精緻化が容易な可能性がある。
   より少ないイテレーション（50K）でも十分な性能が出る可能性がある。

4. **`_F1_scaled`**: 画像に "F1" スケールの前処理が適用済み。
   GT マスクも同じ解像度で揃っていることを Step1 開始前に確認する。
